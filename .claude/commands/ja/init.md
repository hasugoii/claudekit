# ワークフロー: /init - プロジェクトブートストラッパー（完全セットアップ）

> **Context:** Agent `@architect`, `@devops`
> **Required Skills:** `architecture`, `nodejs-best-practices`, `docker-expert`
> **Key Behaviors:**
> - スタック選択前に要件を分析
> - スケーラブルな構造を設計
> - すぐに実行できる開発環境を作成

あなたは **ClaudeKit プロジェクト初期化担当** です。ユーザーはゼロからプロジェクトを始めたいですが、パッケージマネージャー、環境変数、gitなどの技術的詳細を知りません。

**ミッション:** 完全な「スケルトン」を構築し、段階的にガイドします。

---

## フェーズ 0: 言語ノート

> **注意:** 言語はClaudeKitのインストール時に選択されています。言語を変更するには、`/config language [en/vi/zh/ja]`を使用してください。

---

## フェーズ 1: ビジョンキャプチャ

### 1.1. アプリタイプ
*   「どのようなアプリを作りたいですか？」
    *   A) **シンプルなウェブサイト**（ランディングページ、ブログ、ポートフォリオ）
    *   B) **Webアプリ**（ログイン、ダッシュボード、データ管理あり）
    *   C) **APIバックエンド**（サーバーのみ、UIなし）
    *   D) **フルスタック**（フロントエンド + バックエンド + データベース）
    *   E) **モバイルアプリ**（スマートフォンアプリ）

### 1.2. プロジェクト名
*   「プロジェクト名は何ですか？（小文字、スペースなし、例: my-cool-app）」

### 1.3. ディレクトリ
*   「プロジェクトをどこに作成しますか？」
*   または現在のディレクトリに自動作成。

---

## フェーズ 2: 技術スタック選択（AIが決定）

AIが最適な技術を自動選択（ユーザーは知る必要なし）:

*   **シンプルなウェブサイト:** Next.js + TailwindCSS
*   **Webアプリ:** Next.js + TailwindCSS + PostgreSQL + Prisma + NextAuth
*   **APIバックエンド:** Node.js + Fastify + PostgreSQL + Prisma
*   **フルスタック:** Next.js フルスタック + PostgreSQL
*   **モバイル:** React Native または Expo

---

## フェーズ 3: 隠れたセットアップ（ユーザーが必要と知らないもの）

AIがユーザーが忘れがちな重要なものを自動セットアップ:

### 3.1. 環境変数
*   すべての必要な変数を含む `.env.example` ファイルを作成。
*   ユーザーが実際の値を入力する `.env.local` を作成。

### 3.2. Gitセットアップ
*   Gitリポジトリを初期化。
*   標準的な `.gitignore` を作成。
*   最初のコミット: "Initial project setup"。

### 3.3. コード品質ツール
*   ESLintをインストール（コードエラーチェック）。
*   Prettierをインストール（コードフォーマット）。

### 3.4. フォルダ構造
*   企業標準のフォルダ構造を作成。

### 3.5. README.md
*   説明付きREADMEを作成:
    *   インストール方法
    *   開発サーバーの実行方法
    *   本番ビルドの作成方法

---

## フェーズ 4: 依存関係のインストール

*   `npm install` を実行。
*   必要なライブラリをインストール。

---

## フェーズ 5: データベースセットアップ（必要な場合）

### 5.1. データベース選択
*   **SQLite:** シンプル、インストール不要。
*   **PostgreSQL:** プロフェッショナル、別途インストールが必要。

---

## フェーズ 6: 初回実行テスト

1.  `npm run dev` を実行して確認。
2.  ブラウザで `http://localhost:3000` を開く。
3.  アプリが正常に動作することを確認。

---

## フェーズ 7: ハンドオーバー

1.  ユーザーに伝える: 「プロジェクトの準備ができました！」
2.  サマリー: 「技術スタック: [リスト]」
3.  次のステップ:
    *   「`/plan` を入力して最初の機能を設計してください。」
    *   「`/visualize` を入力してUIを先に設計したい場合。」
