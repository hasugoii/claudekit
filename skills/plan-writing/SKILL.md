# Plan Writing

> Viết implementation plans hiệu quả

## Plan Structure

```markdown
# Feature: [Name]

## Overview
[Brief description]

## Requirements
### Functional
- [ ] Requirement 1
- [ ] Requirement 2

### Non-functional
- [ ] Performance
- [ ] Security

## Technical Design
### Database
[Schema changes]

### API
[Endpoints]

### UI
[Components]

## Implementation Steps
1. [ ] Step 1
2. [ ] Step 2

## Timeline
| Phase | Duration |
|-------|----------|
| Design | 1 day |
| Backend | 2 days |
| Frontend | 2 days |
| Testing | 1 day |

## Risks
- Risk 1: [Mitigation]
- Risk 2: [Mitigation]
```

## Tips
- Break into small, testable chunks
- Dependencies first
- Define done criteria
- Include rollback plan
